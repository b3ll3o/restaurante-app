## Fluxo de Trabalho

### Criar Novo PRD

1. Criar arquivo em `backlog/` com nome descritivo
2. Usar template `templates/prd-template.md`
3. Descrever: problema, público-alvo, resultado esperado, critérios de sucesso

### Avaliar PRD

| Decisão | Ação |
|---------|------|
| Aprovado | Criar change em `changes/` e seguir pipeline SDD |
| Rejeitado | Mover para `backlog/archive/` com sufixo `-rejected` |
| Duplicado | Vincular ao PRD original e arquivar |

### Prioridade

| Prioridade | Descrição |
|------------|-----------|
| Crítica | Afeta produção, precisa resolver imediatamente |
| Alta | Impacto alto no negócio ou UX |
| Média | Melhoria importante mas não urgente |
| Baixa | Nice-to-have |
