## 1. Descrição do Erro

A página `/admin/login` ficava em reload infinito ao ser acessada. O navegador recarregava a página repetidamente sem parar, impedindo o usuário de fazer login no painel admin.

## 4. Causa Imediata

`createClient()` era chamado no nível do componente `AdminLayout`, gerando **uma nova instância a cada renderização**. Isso fazia com que `supabase.auth` fosse um objeto diferente a cada render.

## 5. Causa Raiz (5 Porquês)

1. **Por que o useEffect re-executava infinitamente?** → Porque `supabase.auth` era uma dependência que mudava a cada render
2. **Por que `supabase.auth` mudava a cada render?** → Porque `createClient()` era chamado diretamente no corpo do componente
3. **Por que `createClient()` não estava memoizado?** → O código não usava `useMemo` ou `useRef` para estabilizar a referência
4. **Por que o código não seguia o padrão de estabilização?** → Provavelmente esquecido durante implementação do PRD 008
5. **Por que não foi detectado antes?** → Falta de teste E2E para o fluxo de login

## 8. Correção Aplicada

**Arquivo:** `app/admin/layout.tsx`

```tsx
// ANTES (buggy)
const supabase = createClient();

// DEPOIS (corrigido)
const supabase = useMemo(() => createClient(), []);
```

Importação adicionada: `useMemo` de `react`

## 9. Lições Aprendidas

1. `createClient()` do Supabase **DEVE** ser memoizado quando usado em componentes client
2. Dependencies de `useEffect` que são objetos devem ser evitadas
3. Mudanças em `lib/supabase/` podem afetar toda a aplicação - preciso de testes
